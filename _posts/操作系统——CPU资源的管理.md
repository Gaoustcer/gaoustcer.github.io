## 多进程图像

充分利用CPU计算时间，引入多个进程

### 用户和操作系统眼中的多进程

* 对于用户而言，多个运行的程序**同时**在运行，利用计算机系统的资源
* 操作系统负责保存多个程序运行的信息，以及分配计算系统的资源给予多个进程
* 从启动开始到关机结束`if(!fork()) {init();}`创建shell
* shell接收命令创建进程

```c
int main(int argc,char *argv[]){
    while(1){
        scanf("%s",cmd);
        if(!fork()){
            exec(cmd);
            wait();
        }
    }
}
```

### 操作系统内核对多进程的支持——PCB

* PCB用来记录进程信息的数据结构
* pid是进程的标识
* 根据进程状态，进程分为三个部分(进程状态图)
  * 正在执行的进程——运行态
  * 等待被执行的进程——就绪态
  * 等待某事件完成的进程——阻塞态

### 进程交替的实现

* 修改进程状态`pcur.state='W'`
* 将进程放到其它队列中`pcur->waitqueue()`
* 进程切换`Schedule()`->调度算法

```c
void schedule(){
    pnew = getnew(ReadyQueue); // 进程切换本质上是设计getnew函数
    switch_new(pnew)
}
```

switch_new实际上是读取硬件数据并保存到内存结构体中，并将内存数据保存到硬件中

### 多进程通讯/合作

## 用户级线程

### 问题——代价更低的上下文切换

* 进程=资源+指令执行序列
* 不同的进程拥有不同的内存映射表(进程执行相互独立)，如何在切换进程的同时，不切换内存映射表
  * 仅仅切换指令执行序列和保存寄存器
* 分离指令和执行资源，使得一个资源对应多个指令执行序列
  * 线程——保留了并发的特点，同时减少了进程切换的代价

### 线程的价值

* 线程=多个执行序列+一个地址空间
* 浏览器加载一个网页，可能同时产生多个线程
  * 一个线程用于网络通信
  * 一个线程用来处理图片
  * 一个线程用来显示文本
* 有利于线程间共享资源，通过内存
* create->yield->create->yield不断产生线程，切换线程
* 线程拥有独立的**运行时**地址空间

### 何为线程切换？

两个代码执行序列之间的切换

#### 函数调用与线程切换

* 函数调用通过保存和维护返回地址栈实现
* 两个线程使用两个返回地址栈，可以保证切换线程不会导致返回地址错误
* 切换线程首先需要切换栈地址指针
* TCB：线程控制全局数据结构，TCB和栈相互配合实现线程切换

```cpp
void Yield(){
    TCB2.esp = esp;
    esp = TCB1.esp;
    jmp 204; // should be removed,for stack transform
}// 实现线程切换
```

```cpp
void createthread(A){// 切换线程
    TCB *tcb = malloc();
    *stack = malloc();// 申请一个新的线程栈
    *stack = A;// 栈中第一个地址是线程的起始地址
    tcb.esp = stack
}// A是线程的初始地址
```

#### 什么是用户级

* 线程切换由用户控制，实现用户多线程，不管操作系统层的多线程支持，线程之间的切换完全在用户地址进行，线程切换实际上就是改变程序指针的指向
* 线程阻塞(进行I/O，网络通信)的情况
  * 用户级线程对于操作系统内核不可见，操作系统内核只能看到不同进程
  * 某一个线程出现阻塞->导致这个线程在内核中的镜像(进程)阻塞->进程切换->原进程其它产生的线程也无法运行->卡顿，一个用户级线程阻塞导致其它线程也被阻塞
  * 但是内核级线程对于内核是可见的，不会导致一个线程导致其它线程被阻塞的情况

## 内核级线程

### 为什么引入内核级线程的概念

支持多核，多个核公用一套MMU和cache，多个线程运行于多个核上，公用一套内存映射表。仅仅使用用户级线程无法在内核层次分配计算资源

用户级线程无法充分利用多核并行性，因为用户态创建多个线程对于OS而言是透明的，内核不参与用户线程的调度，也不会为用户级线程分配资源

### 内核级线程 vs 核心级线程

* 引入一个系统调用`ThreadCreate()`内核管理TCB和切换线程
* 维护用户栈和核心栈->两套栈
* TCB关联内核栈，切换一套栈

### 用户栈 && 内核栈

* 每个线程创建时分配用户栈和内核栈
* 一旦遇到中断，就切换到内核栈
* 硬件压栈，源SS,SP,PC,CS,EFLAGS连接内核栈和用户栈的媒介(int返回地址)
* 内核级线程的切换本质上也是TCB的切换
* iret指令从内核栈切换到用户栈

### 内核级线程的中断

* 进入中断
* 处理中断过程中导致中断(内核代码阻塞或者时钟中断，分时系统)
* 寻找切换的目的TCB，切换内核栈
* 执行内核代码
* iret切换回用户态

## 内核级线程的代码实现

从fork()开始，创建内核态线程并实现线程切换

```c
int main(){
    B();
    A();
}
A(){
    fork();
}
```

实际上对应的汇编代码为

```assembly
mov %eax,__NR_fork
INT 0x80
mov res,%eax
```

执行int 0x80，CPU找到当前的内核栈，并压栈以下内容

* SS:SP指向用户态栈
* EFLAGS，当前程序运行得到的状态bit
* CS:IP，指向int执行结束后的下一条指令，对应`mov res,%eax`

执行`system_call`，首先将用户态运行时状态压栈，随后调用`sys_fork`

实现调度，在以下条件下触发线程调度

* 线程被阻塞
* 线程运行时间片耗尽

线程调度主要任务是：找到下一个执行的核心级线程和完成核心态的切换

完成核心态的切换——TSS任务结构段，现代操作系统使用kernel切换，操作系统寻找段的方案是根据TR找到TSS描述符，再定位到具体的段，类似于快照
