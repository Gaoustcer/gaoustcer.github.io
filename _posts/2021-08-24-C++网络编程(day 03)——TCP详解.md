# C++网络编程(day 03)——TCP详解

## socket缓冲区以及阻塞模式

本节看看数据如何在两个线程之间传递

### socket缓冲区

在socket API中，我们通过`write()/send()`通过网络传输数据，先将数据写入到缓冲区中，再由TCP协议将数据从缓冲区发送到目标机器。数据写入到缓冲区后，函数就会返回，而不需要等待数据到达目标机器

`read()/recv()`函数也是从缓冲区读取数据，TCP协议独立于缓冲区，数据可能在刚刚进入缓冲区就被发送/读取，也可能在缓冲区积累。缓冲区独立以下特性

* I/O缓冲区在每个TCP套接字单独存在
* I/O缓冲过去在创建套接字自动生成
* 关闭套接字也会继续传送缓冲区遗留的数据
* 关闭套接字将会丢失缓冲区中的数据

输入/输出缓冲区的大小一般为8k，可以根据`getsockopt()`函数获取

```cpp
int getsockopt(int sock, int level, int optname, void *optval, socklen_t *optlen);
```

* sock需要设置或者换取选项的套接字
* level选项所在的协议层
* optname需要访问的选项名
* optval对于`getsockopt()`指向返回选项值的缓冲，对于`setsockopt()`指向包含新选项值的缓冲
* optlen对于`getsockopt()`作为入口参数选项值的最大长度，作为出口参数，选项值的实际长度

### 阻塞模式

TCP套接字在使用`write()/send()`发送数据时，需要经过以下步骤

1. 检查输出缓冲区，如果缓冲区可用空间长度小于待发送的数据，那么`write()/send()`会被阻塞
2. 如果TCP协议正在向网络发送数据，那么输出缓冲区会被加锁，不可写入，write/send会被阻塞。数据发送完毕会向缓冲区发送一个信号解锁缓冲区，`write()/send()`会被唤醒
3. 如果要写入的数据大于缓冲区的最大长度，需要分批写入
4. 所有数据写入缓冲区才能返回

读取数据时

1. 检查缓冲区，如果存在数据就会读取，如果没有数据，函数会被阻塞，知道网络传递新的数据到来
2. 如果要读取的数据长度小于缓冲区的数据长度，则不能一次性将缓冲区所有数据读出，剩余数据在缓冲区中积压
3. 读取数据后`read()/recv()`才会返回

## TCP三次握手和四次挥手

两个主机之前通过TCP建立连接，需要经历三次握手，这是在`connect()`函数中进行的过程，假设socket A要与socket B进行通信，则A和B之间需要传输三个数据包，假设起始的seq分别是1000和2000

### TCP数据报的结构

* 源端口，目的端口
* 序列号(Seq)，确认号(Ack)，均为32位
* 标志位
  * URG 紧急指针有效
  * ACK 确认序号有效
  * PSH 接受方应将这个报文尽快交给应用层
  * RST 重置连接
  * SYN 建立一个新连接
  * FIN 断开一个连接
* 首部长度
* 窗口

### 三次握手，假设A想与B建立连接

* A向B发送 (Seq=1000,SYN)数据包
* B向A发送 (Seq=2000,Ack=1001,SYN+ACK)数据包
* A向B发送 (Ack=2001,ACK)数据包

随后传输数据，当且仅当服务端调用listen进入`LISTEN`状态之后才能相应客户端，此时客户端向服务端发送请求

1. 客户端调用`connect()`函数建立与服务端的连接欸，TCP协议会组件一个数据包并设置SYN标志位，用来同步连接，同时随机选取一个数字1000填充Seq字段，向服务端发送数据包，客户端进入`SYN-SEND`状态
2. 服务端接收到数据包，检测到设置了SYN标志位，这是客户端发送建立连接的**请求包**，服务端组建一个数据包，设置SYN和ACK标志位，SYN表示数据包用来建立连接，ACK确认收到的来自客户端的数据包，Seq字段随机生成，和客户端无关，Ack=客户端Seq+1
3. 客户端收到服务端的确认包，检查确认号字段，如果为1001，则说明连接建立成功

### 数据传输过程

ACK号增量为传输的数据字节数，假设主机A向主机B发送一个大小为1000字节的数据包，起始的seq为1200，则回复的ACK=1301，第二次发送的数据包Seq=1301，返回的ACK=1402，我们有

$Ack=Seq+transformbyte+1$

如果存在数据包丢失的情况，比如A传输的一个大小为100字节的数据包因为网络原因并未到达主机B，则主机A会等待一段时间(RTO,重传超时时间)后会重传这个数据包。

我们一般会为单个数据包设置一个最大重传次数，如果重传达到最大重传数据仍然未能收到ACK，则认为当前网络状态并不适合远程通信，停止通信

### TCP四次挥手，假设A已经传输完所有数据，现在需要断开与B的连接

建立连接后。客户端和服务端都处于`ESTABLISED`状态，客户端发起断开连接的请求

1. 客户端调用close函数，向服务器发送FIN数据包，进入`FIN_WAIT_1`状态，表示完成任务需要断开连接
2. 服务器收到数据包后，检查该数据包设置了FIN标志位，向客户端发送确认包，进入`CLOSE_WAIT`状态
3. 客户收到服务器的确认包进入`FIN_WAIT_2`状态
4. 服务器准备完毕(回收资源)，断开连接，主动向客户端发送FIN保，断开连接，进入`LAST_ACK`
5. 客户端收到服务器的FIN保，向服务器发送ACK包，客户端进入`TIME_WAIT`
6. 服务器收到客户端的ACK包，断开连接，关闭套接字，进入`CLOSED`

客户端进入`TIME_WAIT`之后需要等待2MSL(Maximum Segment Lifetime)之后才会进入closed状态，如果2MSL之后还没有收到服务器重传的FIN包，则说明服务器已经收到了ACK包
